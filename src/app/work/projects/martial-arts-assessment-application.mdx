## Overview

This Digital Scoring System for Pencak Silat is a web-based platform designed to address the inefficiencies and potential inaccuracies of manual scoring in martial arts competitions. By providing a centralized and real-time solution, the application empowers officials to conduct matches with higher transparency and accuracy. It features distinct interfaces for different roles (Admin, Judge, Jury Council, Timekeeper), all synchronized through a live communication channel. The system's primary goal is to digitize the entire scoring workflow, from participant management to the final verdict, ensuring every action is logged and results are calculated automatically.

## Key Features

- **Multi-Role Access Control**: A secure, role-based system ensuring users (Admin, Judge, Jury Council, Timekeeper) only have access to functionalities relevant to their duties.
- **Real-Time Scoring Engine**: Utilizes WebSockets (via Socket.IO) to instantly broadcast scores, penalties, and time status to all connected officials, eliminating delays and discrepancies.
- **Centralized Match Control**: The Jury Council can start, pause, and end matches, while also having the authority to issue penalties that are immediately reflected in the total score.
- **Comprehensive Data Management**: An admin dashboard allows for full CRUD (Create, Read, Update, Delete) operations on participant data and system user accounts.
- **Digital Match Archive & Logging**: Every match's detailed results are automatically saved and can be reviewed later, creating a reliable digital archive for analysis and record-keeping.
- **Synchronized Timekeeping**: A dedicated interface for the Timekeeper to manage match time, perfectly in sync with the Jury Council's commands and visible to all parties.

## Technologies Used

- **Next.js**: A React framework used to build this full-stack application, providing server-side rendering (SSR) for performance and integrated API routes for backend logic.
- **TypeScript**: Ensures type safety and improves code quality across the project, which is critical for a system where data accuracy is paramount.
- **PostgreSQL**: A robust and reliable relational database chosen for storing structured data such as match results, participant details, and user credentials.
- **Prisma**: A modern ORM that enables type-safe and intuitive database access, simplifying queries and migrations between the application and the PostgreSQL database.
- **Socket.IO**: The core engine enabling real-time, bi-directional, and event-based communication that synchronizes the state of the match across all clients.

## Technical Deep Dive: Late-Joiner Synchronization

The most complex challenge in this real-time scoring system was ensuring that a judge or official who joins a match already in progress receives the correct, up-to-the-second state. I implemented an automatic synchronization pattern using **Socket.io** that broadcasts the current match data as soon as a new connection is established.

```typescript
// server/ioServer.ts - Late-joiner synchronization logic
ioServer.on("connection", (socket) => {
    socket.on("joinMatch", (msg) => {
        socket.join(msg.room);
        
        // AUTOMATIC SYNC: When a new user joins, push the current match state 
        // to that specific socket immediately.
        matchSeniController.syncMatch(ioServer, socket);
        matchTandingController.syncMatch(ioServer, socket);
        
        ioServer.to(msg.room).emit(
            "validasiUser",
            `${msg.role} has joined room ${msg.room}.`
        );
    });

    // Special handling for the Timekeeper role
    socket.on("disconnect", () => {
        if (socket.data.role === "timekeeper") {
            // Safety measure: Stop the match clock if the timekeeper drops
            ioServer.to("Match").emit("stopTime");
        }
    });
});
```

### Rationale
In high-stakes sports like Pencak Silat, data consistency is non-negotiable. If a Jury Council's internet blips, this pattern ensures they don't have to manually refresh or ask about the current score—they are re-synced instantly by the backend as soon as they reconnect.

## Challenges and Learnings

Another challenge was designing the database schema with **Prisma and PostgreSQL** to accurately model the complex relationships between matches, participants, judges, individual scores, and penalties. Structuring it for both write-intensive operations during a live match and efficient read operations for post-match analytics was a balancing act.

Through this project, we gained deep expertise in full-stack development with Next.js and the intricacies of WebSocket communication. We learned the importance of designing a scalable and fault-tolerant system architecture from the outset. Ultimately, this project solidified our skills in building mission-critical applications where real-time accuracy and data integrity are not just features, but the core foundation of the product.
