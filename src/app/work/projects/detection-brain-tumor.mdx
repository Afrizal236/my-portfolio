## Overview

This project is a complete deep learning system designed for the multi-class classification of brain tumors from MRI scans. It utilizes a Convolutional Neural Network (CNN) built with Python, TensorFlow, and Keras to accurately categorize images into one of four classes: Glioma, Meningioma, Pituitary, or No Tumor. The model development and training are conducted in a Google Colab environment to leverage cloud GPU acceleration. After training, the model is deployed using a Flask-based REST API, which allows users or other applications to upload a brain scan image and receive an instant prediction, transforming the analytical model into a practical, interactive tool.

## Key Features

- **Multi-Class Brain Tumor Classification:**: Allows users to train detection models for any custom object simply by providing a labeled dataset, creating highly specialized solutions.
- **Deep CNN for High Accuracy**: Implementing the lightweight SSD-MobileNetV2 architecture using TensorFlow, resulting in a high-performance model optimized for real-time detection on edge devices.
- **REST API for Real-Time Prediction**: Deploys the trained model using a Flask web server, exposing a REST API endpoint that allows users to upload an image via an HTTP request and receive an immediate classification result.
- **Specialized Medical Image Processing**: Implements key preprocessing steps, including converting images to grayscale and resizing them to a uniform dimension, to enhance relevant feature extraction and improve model performance.
- **Cloud-Based Model Training**: The entire model training and validation process is managed within a Google Colab notebook, streamlining setup and leveraging powerful cloud GPUs for efficient computation.

## Technologies Used

- **Backend & API: Flask**: Used to create a lightweight web server and expose the prediction model via a REST API.
- **Deep Learning & AI: TensorFlow, Keras**: For building and training the CNN model.
- **Programming Language Python: **: The core language for model development and the Flask server

## Technical Deep Dive: LLM-Ready Prediction Pipelines

While many AI projects end at the model training stage, I designed this system with **LLM Integration** in mind. Instead of returning raw probability tensors, the backend uses a dedicated `Predictor` service that transforms model outputs into structured, JSON-safe context that an LLM (like GPT-4 or Gemini) can "read" to provide diagnostic explanations.

```python
class Predictor:
    def predict(self, image_path: str) -> Dict:
        # 1. Pre-process and Inference
        predictions = self._model.predict(img_array, verbose=0)
        
        # 2. Extract confidence and per-class probabilities
        predicted_index = int(np.argmax(predictions[0]))
        confidence = float(predictions[0][predicted_index])
        
        # 3. Return JSON-safe context for LLM agents
        return {
            "status": "success",
            "predicted_class": self.class_names[predicted_index],
            "confidence": round(confidence, 6),
            "probabilities": {
                name: float(round(predictions[0][i], 6))
                for i, name in enumerate(self.class_names)
            }
        }
```

### Rationale
By decoupling the model inference from the response generation, we can feed the resulting `probabilities` dictionary directly into an LLM's system prompt. This allows the AI to explain *why* it chose a specific class (e.g., "The model shows a 95% confidence for Glioma") and suggest next steps in a human-readable format.

## Challenges and Learnings

The project began with significant challenges in curating a high-quality custom dataset, a process that underscored the core lesson that model performance is fundamentally determined by the quality and diversity of its training data. During the training phase on Google Colab, computational challenges such as out-of-memory errors emerged, which taught us the importance of resource management through tuning hyperparameters such as batch_size to balance stability and speed. Moving into the deployment phase, the challenge shifted to bridging the model from the cloud to constrained edge devices, leading to crucial learnings about post-training optimizations by converting the model to the TensorFlow Lite (TFLite) format to enable efficient inference. Overall, the project emphasized that machine learning development is an iterative process, where achieving optimal solutions is not a straight line, but rather a continuous cycle of experimentation, monitoring, and refinement.
